# BITRIX24 SETUP

## 1. Регистрация и CRM

1. Зарегистрируйтесь на [bitrix24.ru](https://www.bitrix24.ru/).
2. Создайте портал компании.
3. Откройте `CRM`.
4. Включите работу с `Лидами`.
5. Добавьте менеджеров в портал и проверьте, что у них есть доступ к CRM.

## 2. Webhook

1. Откройте `Разработчикам`.
2. Выберите `Другое` -> `Входящий вебхук`.
3. Дайте права `CRM`.
4. Скопируйте URL вида:

```text
https://your-portal.bitrix24.ru/rest/1/xxxxxxxxxxxx
```

## 3. Куда вставить URL

Создайте `.env.local` в корне проекта:

```env
BITRIX_WEBHOOK_URL=https://your-portal.bitrix24.ru/rest/1/xxxxxxxxxxxx
BITRIX_SOURCE=Website
```

Если нужен один ответственный менеджер:

```env
BITRIX_ASSIGNED_BY_ID=1
```

Если нужно распределение между менеджерами:

```env
BITRIX_MANAGER_IDS=1,7,12
```

Сайт выберет менеджера по телефону клиента, чтобы заявки распределялись стабильно между указанными ID. Для строгого round robin нужен внешний backend с хранением счетчика.

## 4. Как проверить заявку

1. Запустите сайт: `npm run dev`.
2. Отправьте тестовую заявку.
3. Откройте `CRM -> Лиды`.
4. Проверьте новый лид: имя, телефон, товар, комментарий.
5. Если включен `BITRIX_ASSIGNED_BY_ID` или `BITRIX_MANAGER_IDS`, проверьте поле ответственного.

## 5. Если webhook не указан

Сайт не падает. API возвращает успешный технический ответ с `crmConfigured: false`, а форма показывает понятное сообщение. Но в Bitrix24 лид не попадет, пока не настроен `BITRIX_WEBHOOK_URL` или внешний endpoint для статического хостинга.

## 6. Как назначать менеджеров

Есть 4 рабочих варианта.

**1. Один ответственный**

```env
BITRIX_ASSIGNED_BY_ID=1
```

Просто и надежно, но все заявки идут одному человеку.

**2. Распределение между менеджерами**

```env
BITRIX_MANAGER_IDS=1,7,12
```

Хороший вариант для старта. Заявки распределяются по списку менеджеров на основе телефона клиента.

**3. Round Robin в Bitrix24**

Лучший вариант для отдела продаж. Настраивается внутри Bitrix24 через роботов, очереди, правила распределения или бизнес-процессы. Сайт создает лид, а Bitrix24 сам назначает следующего менеджера.

**4. Всем менеджерам уведомление**

Через Telegram:

```env
TELEGRAM_BOT_TOKEN=123456:bot-token
TELEGRAM_MANAGER_CHAT_IDS=111111,222222,333333
```

Каждый chat ID получит сообщение о новой заявке.

## 7. Telegram уведомления

1. Создайте бота через `@BotFather`.
2. Получите `TELEGRAM_BOT_TOKEN`.
3. Узнайте chat ID менеджеров или группового чата.
4. Добавьте переменные:

```env
TELEGRAM_BOT_TOKEN=
TELEGRAM_MANAGER_CHAT_IDS=
```

При каждой заявке сайт отправит сообщение с именем, телефоном, товаром и комментарием.

## 8. Email уведомления

Для email лучше использовать внешний сервис: UniSender, SendPulse, Mailgun, SMTP backend или CRM-уведомления Bitrix24. В статическом `out` email напрямую не отправляется, нужен внешний endpoint.

## 9. Как видеть лиды

Откройте `CRM -> Лиды`. Рекомендуемые колонки:

- название лида;
- имя;
- телефон;
- ответственный;
- стадия;
- дата создания;
- источник.

## 10. Что лучше для Septus

Для production запуска оптимально:

1. Сайт создает лид в Bitrix24.
2. Bitrix24 распределяет лиды через очередь/роботов.
3. Telegram-группа менеджеров получает дублирующее уведомление.
4. Email использовать как резерв, если Telegram или Bitrix24 временно недоступны.

Так заявки не теряются и не завязаны на одного менеджера.
