# Sanity: что за что отвечает

Контент для публичного сайта подтягивается из проекта Sanity (`NEXT_PUBLIC_SANITY_PROJECT_ID`, `NEXT_PUBLIC_SANITY_DATASET`). После **Publish** изменения попадают на сайт после сброса кэша Next.js.

## Studio (редактор)

- **Локально:** `npm run studio` → обычно http://localhost:3333  
- **На проде (если собран деплой):** статическая Studio в каталоге **`/admin/`** на домене сайта (см. скрипт `build:studio` в `package.json`).

Доступ в Studio: через [sanity.io/manage](https://www.sanity.io/manage) — приглашение пользователя в проект (роль **Editor** или **Administrator**).

## Документы

| Тип | Назначение |
|-----|------------|
| **Главная страница** (`homePage`, singleton) | Hero (плашка, заголовок, текст, фото, бейджи), кнопки hero, преимущества, инфоблоки «Замер / доставка / оплата», ссылка под ними, FAQ, блок заявки, ссылки на **кейсы на главной** (references). |
| **Каталог — серия / модель** (`product`) | Серия: бренд, slug серии, цены (`price`, `priceValue`, `pricePrefix`, `priceWas`), SEO, фото, характеристики, доп. строки таблицы (`specsExtra`). Массив **models** задаёт отдельные URL моделей (`modelSlug`, люди, сброс, цена текста, при необходимости производительность/залп). Если **models** заполнены — в каталоге только они (без автогенерации из кода). |
| **Кейс / монтаж** (`caseStudy`) | Страницы `/cases/[slug]/`: фото, описание, работы, оборудование (пары ключ–значение), SEO, флаг публикации, порядок. |
| **Акция** (`promotion`) | Страница `/promotions/`: заголовок, текст, бейдж, %, картинка, даты, кнопка, активность, порядок. |
| **Отзыв** (`review`) | Главная — блок отзывов. |
| **Контакты** (`contacts`) | Контактные данные (если используются на сайте). |
| **Глобальное SEO** (`seoSettings`) | Общие SEO-настройки (если подключены в шаблонах). |
| **Прайс-строки** (`priceItem`), **Категории** (`category`) | Справочники / прайс (при необходимости). |

## Публикация и сайт

1. Внесите правки в документ → **Publish**.  
2. Сайт кэширует ответы Sanity (обычно до суток). Чтобы обновление появилось **сразу**, настройте webhook в Sanity:

   - URL: `https://ВАШ_ДОМЕН/api/revalidate?secret=СЕКРЕТ`  
   - Метод: **POST**  
   - Секрет должен совпадать с переменной **`SANITY_REVALIDATE_SECRET`** на хостинге (Vercel / сервер).

3. При успешном запросе API возвращает `{ ok: true }` и сбрасывает тег кэша `sanity-cms`.

### Важно: URL сайта в переменных окружения

На **продакшене** в `.env` / панели хостинга задайте:

`NEXT_PUBLIC_BASE_URL=https://www.septus.ru`

Без завершающего слэша. Если оставить `http://localhost:3002`, в **robots.txt** и части ссылок попадёт localhost (плохо для SEO). В коде добавлена защита: при production-сборке localhost подменяется на `https://www.septus.ru`, но правильный env всё равно нужен для вашего реального домена.

### Проверка webhook вручную

```bash
curl -sS -X POST "https://ВАШ_ДОМЕН/api/revalidate?secret=ВАШ_СЕКРЕТ"
```

Ожидается JSON `{"ok":true,...}`. Без секрета или с неверным секретом — `401`.

## Добавить товар

1. **Каталог — серия / модель** → Create.  
2. Заполните **бренд** (как на сайте) — от него URL `/catalog/{бренд-латиницей}/`.  
3. **slug** серии (латиница).  
4. Цены: число `priceValue`, при необходимости текст `price` и префикс `pricePrefix` («от»).  
5. Чтобы появились **отдельные карточки и страницы моделей**, заполните массив **models**: уникальный `modelSlug`, заголовок, число человек, тип сброса, цена строкой.  
6. Включите **Опубликовано** → Publish → вызовите revalidate (или дождитесь истечения кэша).

## Добавить кейс

**Кейсы / монтаж** → Create → slug, фото, работы, оборудование → Publish.  
Чтобы кейс попал в блок на главной: в singleton **Главная страница** добавьте ссылку в **Кейсы на главной**.

## Добавить акцию

**Акции** → Create → даты, текст, изображение, **Показывать на сайте** → Publish.

## Если в Sanity нет товаров

Каталог и кейсы берутся из **файлов в репозитории** (`src/lib/products.ts`, `src/content/cases.ts`, `src/content/promotions.ts`) как запасной вариант.

Как только в Sanity появится **хотя бы один опубликованный товар**, каталог на сайте строится **только из Sanity** (локальный каталог из кода не смешивается).

## Финальный чеклист закрытия проекта

1. **Прод:** задать `NEXT_PUBLIC_BASE_URL=https://www.septus.ru` (без слэша в конце), пересобрать приложение.
2. **Деплой:** вылить последний `main` (коммит **`33f53d58`** или новее).
3. **После деплоя открыть в браузере:** `/robots.txt`, `/sitemap.xml`, favicon/вкладка, `/admin/`, выборочно страницы с **canonical** и **OG** (просмотр кода страницы / Rich Results), проверить **старые URL** (редиректы, не 404).
4. **Webhook:** Publish в Sanity → изменение на сайте **без** ручного redeploy; в Sanity → Webhooks — доставка **HTTP 200**.
5. **Быстрый обход:** mobile / tablet / desktop, формы, каталог, кейсы, меню.
6. Если **`/sitemap.xml` всё ещё 500** — прислать **stack trace / логи сервера** (runtime при генерации sitemap), разберём отдельно.
7. **Sanity:** структура Studio — ок, менять не обязательно.

После пунктов 1–5 проект считается **закрытым** с точки зрения эксплуатации.
